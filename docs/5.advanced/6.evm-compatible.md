# EVM networks

Most of the examples in this documentation are for Ethereum. But DipDup can index any EVM-compatible network as long as there's enough historical data. This page contains the list of supported EVM-compatible networks and a brief guide on how to configure them.

## Configuring datasources

A typical EVM indexer uses three types of datasources for different purposes, detailed in their respective articles: [abi.etherscan](../3.datasources/1.abi_etherscan.md), [evm.subsquid](../3.datasources/4.evm_subsquid.md), [evm.node](../3.datasources/3.evm_node.md). Here is a brief guide on how to set them up for EVM networks.
Examples have been taken from demo_evm_events from which you can start building your EVM network indexer using:

```shell [Terminal]
dipdup new
```


```yaml
datasources:
  etherscan:
    kind: abi.etherscan
    url: ${ETHERSCAN_API_URL:-https://api.etherscan.io/api}
    api_key: ${ETHERSCAN_API_KEY:-''}

  evm_node:
    kind: evm.node
    url: ${NODE_HTTP_URL:-https://eth-mainnet.g.alchemy.com/v2}/${NODE_API_KEY:-''}
    ws_url: ${NODE_WS_URL:-wss://eth-mainnet.g.alchemy.com/v2}/${NODE_API_KEY:-''}

  subsquid:
    kind: evm.subsquid
    url: ${SUBSQUID_URL:-https://v2.archive.subsquid.io/network/ethereum-mainnet}
    node: evm_node
```

related env

```shell
ETHERSCAN_API_KEY=''
ETHERSCAN_API_URL=https://api.etherscan.io/api
NODE_API_KEY=''
NODE_HTTP_URL=https://eth-mainnet.g.alchemy.com/v2
NODE_WS_URL=wss://eth-mainnet.g.alchemy.com/v2
SUBSQUID_URL=https://v2.archive.subsquid.io/network/ethereum-mainnet
```

### [abi.etherscan](../3.datasources/1.abi_etherscan.md)

Many explorers have Etherscan-like API and it could be used to retrieve ABIs for further types generating using `abi.etherscan` datasource.
You can get a URL for it in the table for your network, for some Etherscan-like explorers you will also need an API key.

```yaml [dipdup.yaml]
  etherscan:
    kind: abi.etherscan
    url: ${ETHERSCAN_API_URL:-https://api.etherscan.io/api}
    api_key: ${ETHERSCAN_API_KEY:-''}
```

When Etherscan-like API is not available, you can use ABI from the JSON file.

Put the JSON file with ABI to `abi/<typename>/abi.json` and run `dipdup init` to generate all necessary types, then you can run your indexer.

### [evm.subsquid](../3.datasources/4.evm_subsquid.md)

To configure `evm.subsquid` datasource, paste `evm.subsquid` URL from tables below or set `SUBSQUID_URL` variable in .env file.

```yaml [demo_evm_events/dipdup.yaml]
datasources:
  ...
  subsquid:
    kind: evm.subsquid
    url: ${SUBSQUID_URL:-https://v2.archive.subsquid.io/network/ethereum-mainnet}
    node: evm_node
  ...
```

```yaml [arbitrum/dipdup.yaml]
    ...
    url: https://v2.archive.subsquid.io/network/arbitrum-one
    ...
```

### [evm.node](../3.datasources/3.evm_node.md)

Node RPC API vary a lot across EVM networks, yet most of them supported by dipdup indexer to retrieve historical data.
Public node providers usually require API key, so demo_evm_events template allows you to pass HTTP URL, Websocket url, and API key into NODE_HTTP_URL, NODE_WS_URL, and NODE_API_KEY using .env file to use node providers like Alchemy and Infura from the box.

```yaml [dipdup.yaml]
datasources:
  evm_node:
    kind: evm.node
    url: ${NODE_HTTP_URL:-https://eth-mainnet.g.alchemy.com/v2}/${NODE_API_KEY:-''}
    ws_url: ${NODE_WEBSOCKET_URL:-wss://eth-mainnet.g.alchemy.com/v2}/${NODE_API_KEY:-''}
```
